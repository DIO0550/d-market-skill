# Session Skill Sketch

複数セッションのログを横断的に分析し、次に作るべきスキルの作成計画（スケッチ）を立てる。

---

## 位置付け

- **このスキル**: 複数セッション横断で繰り返しパターンを見つけてスキル化計画を立てる
- **conversation-skill-draft**: 単一セッションを深掘りして具体的なスキル draft を起こす

---

## データソース

`${CLAUDE_PROJECT_DIR}/.claude/plugin-workspace/sessions/<session_id>/` 配下のファイル:

- `session-log.md` — Stopフックで書き出されたセッション要約 (AI生成)
- `user-prompts.jsonl` — UserPromptSubmitフックで記録されたユーザー発言の生ログ
- `approved-commands.jsonl` — PermissionRequestフックで記録された承認コマンドの生ログ

### session-log.md の構造

```markdown
# Session Log

- **Session ID**: xxx
- **Timestamp**: 2025-01-15T10:30:00Z

## 目的・概要
## 使用したツール・コマンド
## やり取りの流れ
## 繰り返しパターン・注目点
## 詰まった箇所・裁量補完
```

### user-prompts.jsonl の構造

1行1発言。`.prompt` にユーザーの発言原文。

### approved-commands.jsonl の構造

1行1イベント。`.tool_name` / `.tool_input` に承認対象。

---

## ワークフロー

### Step 1: セッション一覧の作成

```bash
find .claude/plugin-workspace/sessions -mindepth 1 -maxdepth 1 -type d
```

**ファイルが存在しない場合:**
- まだセッションログが記録されていない旨を伝える
- skill-grower プラグインの hooks (UserPromptSubmit / PermissionRequest / Stop) が有効か確認を促す
- 最低 3 セッション以上のログが蓄積されてからの利用を推奨する

**セッション数 < 3 の場合:**
- 「サンプルが少ないためスケッチの信頼性は低い」と明示して続行する
- 停止はしない（1-2 セッションでも有用な発見はある）

### Step 2: 各セッションの前処理

セッションごとに以下を抽出:

| 項目 | ソース | 内容 |
|---|---|---|
| 目的 | session-log.md | `## 目的・概要` セクション |
| ワークフロー | session-log.md | `## やり取りの流れ` セクション |
| 繰り返しパターン | session-log.md | `## 繰り返しパターン・注目点` セクション |
| 裁量補完 | session-log.md | `## 詰まった箇所・裁量補完` セクション |
| ユーザー発言 | user-prompts.jsonl | 全 prompt の配列 |
| 承認コマンド | approved-commands.jsonl | tool_name / tool_input の配列 |

**session-log.md が期待フォーマットと異なる場合:**
- 欠けているセクションは空として扱い、どのセッションの何が欠けていたかを最終報告に含める
- 無理に解釈して埋めない

### Step 3: 多角的分析

#### 3-1. 目的パターンの分析
- セッションの目的を短いラベル化 (例: "バグ修正", "テスト追加")
- 同一ラベルが複数セッションで現れるものを優先

#### 3-2. ワークフローパターンの分析
- 複数セッションで繰り返される作業の連鎖 (例: 調査 → 実装 → テスト → lint)
- 毎回同じ順序のステップ

#### 3-3. ツール使用パターンの分析
- 頻出するツール・コマンドの組み合わせ
- approved-commands.jsonl から統計的に抽出

#### 3-4. ユーザー訂正パターンの分析 【重要】

user-prompts.jsonl から以下の発言を特定する:

- 方向転換マーカー: 「いや」「違う」「そうじゃなくて」「やっぱり」「待って」
- 追加要件マーカー: 「あと」「さらに」「も」
- 明示の前提共有: 「このプロジェクトでは」「前回は」「いつもは」

これらは「スキル本文に最初から書いておくべきだった暗黙知」の直接の証拠。最優先でスキル化候補にする。

#### 3-5. 繰り返しパターンの横断分析
- 各セッションの `## 繰り返しパターン・注目点` を横断集約
- 複数セッションで共通して指摘されているものを優先

### Step 4: スキル作成計画（スケッチ）の作成

各スケッチに以下を含める:

- **スキル名**: kebab-case の具体名
- **カテゴリ**: ワークフロー自動化 / 品質チェック / プロジェクト管理 / コード生成 / 前提共有 / その他
- **解決する課題**: 毎回手動で行っている作業、または毎回訂正が必要な暗黙知
- **スキルの内容**: 具体的なステップ
- **根拠**: 導出元のセッション ID と該当箇所 (session-log.md の該当セクション引用 or user-prompts.jsonl の該当発言)
- **優先度**: 下記ルール
- **スキルの種類**: 下記判断基準

**優先度の算出ルール:**
- **高**: 3 セッション以上で出現 かつ ユーザー訂正マーカーが 1 回以上含まれる
- **中**: 3 セッション以上で出現、または 2 セッションでワークフロー連鎖として出現
- **低**: 2 セッションのみだが構造的に繰り返しと判断できるもの

**スキルの種類の判断基準:**
- **instruction-only**: 判断ルール・前提共有・ワークフローの説明が主。スクリプト不要
- **with-scripts**: 定型的なコマンド実行・ファイル生成が含まれる
- **with-hooks**: 特定イベント (ファイル保存時など) をトリガーに自動実行したい

### Step 5: ユーザーとの対話

スケッチを提示した上で:
1. 「どのスキルを作りますか？」と聞く
2. ユーザーが選んだスキルについて、`conversation-skill-draft` があれば委譲、無ければ SKILL.md ドラフトを直接作成
3. skill-creator プラグイン (skill-authoring-guide) があれば連携

## スケッチの出力例

```
## セッションログ分析結果

**分析対象**: 12セッション（2025-01-10 〜 2025-01-20）
**欠損**: session abc123 の user-prompts.jsonl が空

---

### スキル作成スケッチ

#### 1. pr-review-workflow（優先度: 高）
- **カテゴリ**: ワークフロー自動化
- **課題**: PRレビューの際に毎回「diff確認 → コード読解 → テスト実行 → コメント作成」を手動で繰り返している
- **内容**: PR番号を指定するだけで、差分の要約・影響範囲の分析・テスト実行・レビューコメントのドラフト作成を一括実行するスキル
- **根拠**: セッション abc123, def456, ghi789 で同一のワークフロー連鎖を検出
  - ユーザー訂正: 「いや、テスト先に走らせて」(session def456)
- **種類**: instruction-only

#### 2. project-convention-context（優先度: 高）
- **カテゴリ**: 前提共有
- **課題**: ユーザーが毎回「このプロジェクトでは Result 型でエラーを返す」「TypeScript は strict 設定」など同じ前提を説明している
- **内容**: プロジェクトのコーディング規約・型の流儀・テスト慣習を最初から知っているスキル
- **根拠**: セッション abc123, ghi789, jkl012 の user-prompts.jsonl で「このプロジェクトでは」が計5回
- **種類**: instruction-only

#### 3. test-and-lint（優先度: 中）
- **カテゴリ**: 品質チェック
- **課題**: 実装後に毎回「テスト実行 → lint → 修正 → 再テスト」のサイクル
- **内容**: テスト＋lint を一括実行、問題があれば自動修正
- **根拠**: 全12セッション中9セッションで繰り返しパターンに記載
- **種類**: with-scripts
```

---

## 前提条件

- `.claude/plugin-workspace/sessions/` に 1 つ以上のセッションディレクトリが存在すること
- 精度の高いスケッチには最低 3 セッション以上推奨
- skill-creator プラグインがあるとスケッチからスキル作成までシームレスに連携可能

## セキュリティ上の注意

user-prompts.jsonl と approved-commands.jsonl は生のユーザー入力・コマンドを含む。
出力に引用する際は、API キー・token・パスワード・メールアドレスなどが混入していないか確認し、検出時はマスクする。
