# Session Skill Sketch

セッションログを分析し、次に作るべきスキルの作成計画（スケッチ）を立てる。

---

## データソース

`.claude/plugin-workspace/session-logs/` 配下のMarkdownファイル。
各ファイルはStopフック（`export-session-log.sh`）によってセッション終了時にAIが書き出したもの。

### ログの構造

```markdown
# Session Log

- **Session ID**: xxx
- **Timestamp**: 2025-01-15T10:30:00Z

## 目的・概要
セッションの目的の要約。

## 使用したツール・コマンド
- Bash: `npm test`
- Edit: src/components/Button.tsx
- ...

## やり取りの流れ
1. ユーザーが○○を依頼
2. △△を調査
3. □□を実装
...

## 繰り返しパターン・注目点
- 毎回テスト実行後にlintを走らせている
- ...
```

---

## ワークフロー

### Step 1: ログの読み込み

`.claude/plugin-workspace/session-logs/` 配下の全`.md`ファイルを読み込む。

ファイルが存在しない場合:
- まだセッションログが記録されていない旨を伝える
- skill-growerプラグインのhooks（Stopフック）が有効になっているか確認を促す
- 最低3セッション以上のログが蓄積されてからの利用を推奨する

### Step 2: 多角的分析

読み込んだログから以下の観点で分析する:

#### 2-1. 目的パターンの分析
- 複数セッションで繰り返される作業目的（例: バグ修正、テスト追加、リファクタリング）
- 似たような目的のセッションで共通する作業フロー

#### 2-2. ワークフローパターンの分析
- 複数セッションで繰り返される作業の流れ（例: 調査 → 実装 → テスト → lint）
- 毎回同じ順序で実行されるステップの連鎖

#### 2-3. ツール使用パターンの分析
- 頻出するツール・コマンドの組み合わせ
- 特定の作業で常に使われるツールセット

#### 2-4. 繰り返しパターンの横断分析
- 各セッションの「繰り返しパターン・注目点」セクションを横断的に集約
- 複数セッションで共通して指摘されているパターン

#### 2-5. コマンドログとの統合分析（任意）
- `.claude/plugin-workspace/approved-commands/` にコマンドログがある場合、セッションログと組み合わせて分析する
- コマンドの頻度データとセッションの文脈を組み合わせることで、より精度の高い提案が可能

### Step 3: スキル作成計画（スケッチ）の作成

分析結果を元に、具体的なスキル作成計画（スケッチ）を作成する。各スケッチには以下を含める:

- **スキル名**: kebab-caseの具体的な名前
- **カテゴリ**: ワークフロー自動化 / 品質チェック / プロジェクト管理 / コード生成 / その他
- **解決する課題**: 毎回手動で行っている作業が何か
- **スキルの内容**: そのスキルが何をするか（具体的なステップ）
- **根拠**: どのセッションログのどのパターンから導出したか（実データを引用）
- **優先度**: 頻度・手間・効果から判断した優先度（高/中/低）
- **スキルの種類**: instruction-only / with-scripts / with-hooks のどれが適切か

### Step 4: ユーザーとの対話

スケッチを見せた上で:
1. 「どのスキルを作りますか？」と聞く
2. ユーザーが選んだスキルについて、SKILL.mdのドラフトを作成する
3. skill-creator プラグイン（skill-authoring-guide）があれば連携してスキルを作成する
4. 必要に応じてhooksやスクリプトの設計も含める

## スケッチの出力例

```
## セッションログ分析結果

**分析対象**: 12セッション（2025-01-10 〜 2025-01-20）

---

### スキル作成スケッチ

#### 1. pr-review-workflow（優先度: 高）
- **カテゴリ**: ワークフロー自動化
- **課題**: PRレビューの際に毎回「diff確認 → コード読解 → テスト実行 → コメント作成」を手動で繰り返している
- **内容**: PR番号を指定するだけで、差分の要約・影響範囲の分析・テスト実行・レビューコメントのドラフト作成を一括実行するスキル
- **根拠**: セッション #3, #7, #11 で同一のワークフローを検出
- **種類**: instruction-only

#### 2. test-and-lint（優先度: 高）
- **カテゴリ**: 品質チェック
- **課題**: 実装後に毎回「テスト実行 → lint → 修正 → 再テスト」のサイクルを手動で回している
- **内容**: テスト＋lint を一括実行し、問題があれば自動修正を試みるスキル
- **根拠**: 全12セッション中9セッションで「繰り返しパターン」に記載あり
- **種類**: with-scripts

#### 3. new-component-scaffold（優先度: 中）
- **カテゴリ**: コード生成
- **課題**: 新しいコンポーネント作成時に毎回同じファイル構成（コンポーネント本体・テスト・ストーリー）を手動で作っている
- **内容**: コンポーネント名を指定するだけで、プロジェクトの規約に沿ったファイル一式を生成するスキル
- **根拠**: セッション #4, #8 でほぼ同一のファイル作成パターンを検出
- **種類**: with-scripts
```

---

## 前提条件

- `.claude/plugin-workspace/session-logs/` にセッションログが存在すること
- 精度の高いスケッチには最低3セッション以上のログが推奨
- skill-creatorプラグインがあると、スケッチからスキル作成までシームレスに連携可能
